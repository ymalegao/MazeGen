Maze Generator

By Yash Malegaonkar, Jaren Kawai, Rahamat Zaman, Chase Houske, Justin Park

Project Overview

Our project is a maze generator. Every time you run the project there is a unique maze that is generated, and the player always spawns in a starter position every time, the player being a cube. Using wasd the player can be moved, and using the arrow keys the camera can be moved. The player can move around the maze and can collect the pellets. You have to watch out for the AI that will hurt you if you touch them.

Each created maze creates paths that both the player and the enemy can take, and like Pac-Man, the paths are lined with pellets that contribute to an overall scoring system. To create new mazes, the generate button must be clicked and a layout will be constructed. The pellets and characters will be reset as well.

Project Goals

The goal of our project is to have a playable example of generation mechanics that were taught throughout this class. We wanted our project to be able to have a player character with movement, that can collect some sort of treasure and escape from an enemy, all while traversing a procedurally generated maze.

Game Mechanics and Features

  • Player (Blue) has 3 Lives
  • Press the generate button to generate new maze outline
    • Able to generate at any given time
    • New maze color (walls)
    • Resets player position
  • Can collect pellets spread all across the maze
    • Game keeps track of pellets collected as score
    • If a power pellet is collected, the AI goes into “fear mode”
  • AI agent (Red) moves around the maze
    • If player touches agent, player loses a life
  • Change camera angle by pressing 1, 2, or 3
